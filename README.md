<p><strong>Julia</strong>&nbsp;is a&nbsp;<a title="High-level programming language" href="https://en.wikipedia.org/wiki/High-level_programming_language">high-level</a>, high-performance, dynamic&nbsp;<a title="Programming language" href="https://en.wikipedia.org/wiki/Programming_language">programming language</a>. While it is a general purpose language and can be used to write any application, many of its features are well-suited for high-performance&nbsp;<a title="Numerical analysis" href="https://en.wikipedia.org/wiki/Numerical_analysis">numerical analysis</a>&nbsp;and&nbsp;<a title="Computational science" href="https://en.wikipedia.org/wiki/Computational_science">computational science</a>.<sup id="cite_ref-17" class="reference"></sup></p>
<p>Distinctive aspects of Julia's design include a type system with&nbsp;<a title="Parametric polymorphism" href="https://en.wikipedia.org/wiki/Parametric_polymorphism">parametric polymorphism</a>&nbsp;in a&nbsp;<a title="Dynamic programming language" href="https://en.wikipedia.org/wiki/Dynamic_programming_language">dynamic programming language</a>; with&nbsp;<a title="Multiple dispatch" href="https://en.wikipedia.org/wiki/Multiple_dispatch">multiple dispatch</a>&nbsp;as its core&nbsp;<a title="Programming paradigm" href="https://en.wikipedia.org/wiki/Programming_paradigm">programming paradigm</a>. Julia supports&nbsp;<a title="Concurrent computing" href="https://en.wikipedia.org/wiki/Concurrent_computing">concurrent</a>, (composable)&nbsp;<a title="Parallel computing" href="https://en.wikipedia.org/wiki/Parallel_computing">parallel</a>&nbsp;and&nbsp;<a title="Distributed computing" href="https://en.wikipedia.org/wiki/Distributed_computing">distributed computing</a>&nbsp;(with or without using&nbsp;<a title="Message Passing Interface" href="https://en.wikipedia.org/wiki/Message_Passing_Interface">MPI</a>&nbsp;and/or the built-in corresponding&nbsp;to "<a title="OpenMP" href="https://en.wikipedia.org/wiki/OpenMP">OpenMP</a>-style" threads), and&nbsp;<a title="Foreign function interface" href="https://en.wikipedia.org/wiki/Foreign_function_interface">direct calling</a>&nbsp;of&nbsp;<a title="C (programming language)" href="https://en.wikipedia.org/wiki/C_(programming_language)">C</a>&nbsp;and&nbsp;<a title="Fortran" href="https://en.wikipedia.org/wiki/Fortran">Fortran</a>&nbsp;libraries without&nbsp;<a title="Adapter pattern" href="https://en.wikipedia.org/wiki/Adapter_pattern#gluecode">glue code</a>. A&nbsp;<a title="Just-in-time compilation" href="https://en.wikipedia.org/wiki/Just-in-time_compilation">just-in-time</a>&nbsp;compiler that is referred to as "just-<a title="Ahead-of-time compilation" href="https://en.wikipedia.org/wiki/Ahead-of-time_compilation">ahead-of-time</a>"&nbsp;in the Julia community is used.</p>
<p>Julia is&nbsp;<a title="Garbage collection (computer science)" href="https://en.wikipedia.org/wiki/Garbage_collection_(computer_science)">garbage-collected</a>,&nbsp;uses&nbsp;<a title="Eager evaluation" href="https://en.wikipedia.org/wiki/Eager_evaluation">eager evaluation</a>, and includes efficient libraries for&nbsp;<a class="mw-redirect" title="Floating-point" href="https://en.wikipedia.org/wiki/Floating-point">floating-point</a>&nbsp;calculations,&nbsp;<a title="Linear algebra" href="https://en.wikipedia.org/wiki/Linear_algebra">linear algebra</a>,&nbsp;<a title="Random number generation" href="https://en.wikipedia.org/wiki/Random_number_generation">random number generation</a>, and&nbsp;<a title="Regular expression" href="https://en.wikipedia.org/wiki/Regular_expression">regular expression</a>&nbsp;matching. Many libraries are available, including some (e.g., for&nbsp;<a title="Fast Fourier transform" href="https://en.wikipedia.org/wiki/Fast_Fourier_transform">fast Fourier transforms</a>) that were previously bundled with Julia and are now separate.</p>
<p>Tools available for Julia include, not just many widely used editors, such as&nbsp;<a title="Vim (text editor)" href="https://en.wikipedia.org/wiki/Vim_(text_editor)">Vim</a>&nbsp;and&nbsp;<a title="Emacs" href="https://en.wikipedia.org/wiki/Emacs">Emacs</a>, but also&nbsp;<a title="Integrated development environment" href="https://en.wikipedia.org/wiki/Integrated_development_environment">IDEs</a>&nbsp;(e.g. Juno, and Microsoft's&nbsp;<a title="Visual Studio Code" href="https://en.wikipedia.org/wiki/Visual_Studio_Code">Visual Studio Code</a>, with&nbsp;<a title="Plug-in (computing)" href="https://en.wikipedia.org/wiki/Plug-in_(computing)">extensions</a>&nbsp;adding Julia support, both providing debugging support); with integrated tools, e.g. a&nbsp;<a title="Lint (software)" href="https://en.wikipedia.org/wiki/Lint_(software)">linter</a>,&nbsp;<a title="Profiling (computer programming)" href="https://en.wikipedia.org/wiki/Profiling_(computer_programming)">profiler</a>&nbsp;(and flame graph support available&nbsp;for the built-in one), debugger,&nbsp;and the Rebugger.jl package "supports&nbsp;<a title="Interactive programming" href="https://en.wikipedia.org/wiki/Interactive_programming">repeated-execution debugging</a>"&nbsp;and more.</p>

<h2><span id="History" class="mw-headline">History</span></h2>
<p>Work on Julia was started in 2009, by Jeff Bezanson,&nbsp;<a title="Stefan Karpinski" href="https://en.wikipedia.org/wiki/Stefan_Karpinski">Stefan Karpinski</a>,&nbsp;<a title="Viral B. Shah" href="https://en.wikipedia.org/wiki/Viral_B._Shah">Viral B. Shah</a>, and&nbsp;<a title="Alan Edelman" href="https://en.wikipedia.org/wiki/Alan_Edelman">Alan Edelman</a>, who set out to create a free language that was both high-level and fast. On 14 February 2012, the team launched a website with a blog post explaining the language's mission.&nbsp;In an interview with InfoWorld in April 2012, Karpinski said of the name "Julia": "There's no good reason, really. It just seemed like a pretty name."&nbsp;Bezanson said he chose the name on the recommendation of a friend.</p>
<p>Since the 2012 launch, the Julia community has grown, with over 13,000,000 downloads as of March&nbsp;2020&nbsp;(and is used at more than 1,500 universities),&nbsp;The Official Julia Docker images, at&nbsp;<a title="Docker, Inc." href="https://en.wikipedia.org/wiki/Docker,_Inc.">Docker Hub</a>, have seen over 4,000,000 downloads as of January 2019.&nbsp;The JuliaCon&nbsp;<a title="Academic conference" href="https://en.wikipedia.org/wiki/Academic_conference">academic conference</a>&nbsp;for Julia users and developers has been held annually since 2014.</p>
<p>Version 0.3 was released in August 2014, version 0.4 in October 2015, version 0.5 in October 2016,&nbsp;and version 0.6 in June 2017.&nbsp;Both Julia 0.7 (a useful release for testing packages, and for knowing how to upgrade them for 1.0) and version 1.0 were released on 8 August 2018. Work on Julia 0.7 was a "huge undertaking" (e.g., because of "entirely new optimizer"), and some changes were made to semantics, e.g. the&nbsp;<a title="Iteration" href="https://en.wikipedia.org/wiki/Iteration">iteration</a>&nbsp;interface was simplified;&nbsp;and the syntax changed a little (with the syntax now stable, and same for 1.x and 0.7).</p>
<p>The release candidate for Julia 1.0 (Julia 1.0.0-rc1) was released on 7 August 2018, and the final version a day later (and by now, Julia 1.0.x are the oldest versions still supported, having&nbsp;<a title="Long-term support" href="https://en.wikipedia.org/wiki/Long-term_support">long-term support</a>; for at least a year). Julia 1.1 was released in January 2019 with, e.g., a new "exception stack" language feature. Bugfix releases are expected roughly monthly, for 1.4.x and 1.0.x and Julia 1.0.1 up to 1.0.5 have followed that schedule. Julia 1.2 was released in August 2019, and it has e.g. some built-in support for web browsers (for testing if running in&nbsp;<a title="JavaScript" href="https://en.wikipedia.org/wiki/JavaScript">JavaScript</a>&nbsp;VM),&nbsp;and Julia 1.3 in November 2019 (and with it Julia 1.2.x, in addition to Julia 1.1.x, releases are no longer maintained), adding e.g. composable multi-threaded parallelism and a binaray artifacts system for Julia packages.&nbsp;JuliaPro version 1.4.0-1 is also out.</p>
<p>Most packages that work in Julia 1.0.x also work in 1.1.x or newer, enabled by the&nbsp;<a title="Forward compatibility" href="https://en.wikipedia.org/wiki/Forward_compatibility">forward compatible</a>&nbsp;syntax guarantee. A major exception was, for interacting with non-Julia code, the JavaCall.jl package (however calling other languages, e.g.&nbsp;<a title="R (programming language)" href="https://en.wikipedia.org/wiki/R_(programming_language)">R language</a>&nbsp;works, with the package for R fixed) to call Java, Scala etc. This was fixed by Java 11,&nbsp;or alternatively to use those languages with Julia (on older JVM), for e.g.&nbsp;<a title="Java Database Connectivity" href="https://en.wikipedia.org/wiki/Java_Database_Connectivity">JDBC</a>.jl or&nbsp;<a title="Apache Spark" href="https://en.wikipedia.org/wiki/Apache_Spark">Apache Spark</a>&nbsp;(through Spark.jl), users could choose to stay with the LTS version of Julia.&nbsp;Julia 1.4 had a milestone set for 15 December 2019 and for Julia 1.5 the due date is 15 April 2020. Milestones for Julia 2.0 (and later, e.g. 3.0) currently have no set due dates.</p>
<h3><span id="Notable_uses" class="mw-headline">Notable uses</span></h3>
<p>Julia has attracted some high-profile users, from investment manager&nbsp;<a title="BlackRock" href="https://en.wikipedia.org/wiki/BlackRock">BlackRock</a>, which uses it for&nbsp;<a class="mw-redirect" title="Time-series analytics" href="https://en.wikipedia.org/wiki/Time-series_analytics">time-series analytics</a>, to the British insurer&nbsp;<a title="Aviva" href="https://en.wikipedia.org/wiki/Aviva">Aviva</a>, which uses it for&nbsp;<a class="mw-redirect" title="Risk calculations" href="https://en.wikipedia.org/wiki/Risk_calculations">risk calculations</a>. In 2015, the&nbsp;<a title="Federal Reserve Bank of New York" href="https://en.wikipedia.org/wiki/Federal_Reserve_Bank_of_New_York">Federal Reserve Bank of New York</a>&nbsp;used Julia to make models of the US economy, noting that the language made model estimation "about 10 times faster" than its previous&nbsp;<a title="MATLAB" href="https://en.wikipedia.org/wiki/MATLAB">MATLAB</a>&nbsp;implementation. Julia's co-founders established Julia Computing in 2015 to provide paid support, training, and consulting services to clients, though Julia itself remains free to use. At the 2017 JuliaCon&nbsp;conference, Jeffrey Regier,&nbsp;<a class="new" title="Keno Fischer (page does not exist)" href="https://en.wikipedia.org/w/index.php?title=Keno_Fischer&amp;action=edit&amp;redlink=1">Keno Fischer</a>&nbsp;and others announced&nbsp;that the Celeste project&nbsp;used Julia to achieve "peak performance of 1.54&nbsp;<a title="FLOPS" href="https://en.wikipedia.org/wiki/FLOPS">petaFLOPS</a>&nbsp;using 1.3 million threads" on 9300&nbsp;<a class="mw-redirect" title="Knights Landing (microarchitecture)" href="https://en.wikipedia.org/wiki/Knights_Landing_(microarchitecture)">Knights Landing</a>&nbsp;(KNL) nodes of the&nbsp;<a title="Cray XC40" href="https://en.wikipedia.org/wiki/Cray_XC40#United_States">Cori II (Cray XC40)</a>&nbsp;supercomputer (then 6th fastest computer in the world).&nbsp;Julia thus joins C, C++, and Fortran as high-level languages in which petaFLOPS computations have been achieved.</p>
<p>Three of the Julia co-creators are the recipients of the 2019&nbsp;<a title="J. H. Wilkinson Prize for Numerical Software" href="https://en.wikipedia.org/wiki/J._H._Wilkinson_Prize_for_Numerical_Software">James H. Wilkinson Prize for Numerical Software</a>&nbsp;(awarded every four years) "for the creation of Julia, an innovative environment for the creation of high-performance tools that enable the analysis and solution of computational science problems."&nbsp;Also, Alan Edelman, professor of applied mathematics at MIT, has been selected to receive the 2019 IEEE Computer Society Sidney Fernbach Award "for outstanding breakthroughs in high-performance computing, linear algebra, and computational science and for contributions to the Julia programming language."</p>
<p>Julia Computing and&nbsp;<a class="mw-redirect" title="NVIDIA" href="https://en.wikipedia.org/wiki/NVIDIA">NVIDIA</a>&nbsp;announce "the availability of the Julia programming language as a pre-packaged container on the NVIDIA GPU Cloud (NGC) container registry"&nbsp;with NVIDIA stating "Easily Deploy Julia on x86 and Arm [..] Julia offers a package for a comprehensive HPC ecosystem covering machine learning, data science, various scientific domains and visualization."</p>
<p>Additionally, "Julia was selected by the&nbsp;<a class="external text" href="https://clima.caltech.edu/" rel="nofollow">Climate Modeling Alliance</a>&nbsp;as the sole implementation language for their next generation global climate model. This multi-million dollar project aims to build an earth-scale climate model providing insight into the effects and challenges of climate change."</p>
<h3><span id="Sponsors" class="mw-headline">Sponsors</span></h3>
<p>Julia has received contributions from over 870 developers worldwide.&nbsp;Dr. Jeremy Kepner at&nbsp;<a title="MIT Lincoln Laboratory" href="https://en.wikipedia.org/wiki/MIT_Lincoln_Laboratory">MIT Lincoln Laboratory</a>&nbsp;was the founding sponsor of the Julia project in its early days. In addition, funds from the&nbsp;<a title="Gordon and Betty Moore Foundation" href="https://en.wikipedia.org/wiki/Gordon_and_Betty_Moore_Foundation">Gordon and Betty Moore Foundation</a>, the&nbsp;<a title="Alfred P. Sloan Foundation" href="https://en.wikipedia.org/wiki/Alfred_P._Sloan_Foundation">Alfred P. Sloan Foundation</a>,&nbsp;<a title="Intel" href="https://en.wikipedia.org/wiki/Intel">Intel</a>, and agencies such as&nbsp;<a title="National Science Foundation" href="https://en.wikipedia.org/wiki/National_Science_Foundation">NSF</a>,&nbsp;<a title="DARPA" href="https://en.wikipedia.org/wiki/DARPA">DARPA</a>,&nbsp;<a title="National Institutes of Health" href="https://en.wikipedia.org/wiki/National_Institutes_of_Health">NIH</a>,&nbsp;<a title="NASA" href="https://en.wikipedia.org/wiki/NASA">NASA</a>, and&nbsp;<a title="Federal Aviation Administration" href="https://en.wikipedia.org/wiki/Federal_Aviation_Administration">FAA</a>&nbsp;have been essential to the development of Julia.&nbsp;In addition&nbsp;<a title="Mozilla" href="https://en.wikipedia.org/wiki/Mozilla">Mozilla</a>, the maker of Firefox web browser, with its research grants for H1 2019, sponsored "a member of the official Julia team" for the project "Bringing Julia to the Browser",&nbsp;meaning to Firefox and other web browsers.</p>
<h3><span id="Julia_Computing" class="mw-headline">Julia Computing</span></h3>
<p>Julia Computing, Inc. was founded in 2015 by&nbsp;<a title="Viral B. Shah" href="https://en.wikipedia.org/wiki/Viral_B._Shah">Viral B. Shah</a>, Deepak Vinchhi,&nbsp;<a title="Alan Edelman" href="https://en.wikipedia.org/wiki/Alan_Edelman">Alan Edelman</a>, Jeff Bezanson,&nbsp;<a title="Stefan Karpinski" href="https://en.wikipedia.org/wiki/Stefan_Karpinski">Stefan Karpinski</a>&nbsp;and Keno Fischer.</p>
<p>In June 2017, Julia Computing raised $4.6M in seed funding from&nbsp;<a title="General Catalyst" href="https://en.wikipedia.org/wiki/General_Catalyst">General Catalyst</a>&nbsp;and&nbsp;<a class="new" title="Founder Collective (page does not exist)" href="https://en.wikipedia.org/w/index.php?title=Founder_Collective&amp;action=edit&amp;redlink=1">Founder Collective</a>.</p>
<h2><span id="Language_features" class="mw-headline">Language features</span></h2>
<p>Though designed for numerical computing, Julia is a&nbsp;<a title="General-purpose programming language" href="https://en.wikipedia.org/wiki/General-purpose_programming_language">general-purpose programming language</a>.&nbsp;It is also useful for low-level&nbsp;<a class="mw-redirect" title="System programming" href="https://en.wikipedia.org/wiki/System_programming">systems programming</a>,&nbsp;as a&nbsp;<a title="Specification language" href="https://en.wikipedia.org/wiki/Specification_language">specification language</a>,&nbsp;and for web programming at both server&nbsp;and client&nbsp;side.</p>
<p>According to the official website, the main features of the language are:</p>
<ul>
<li><a title="Multiple dispatch" href="https://en.wikipedia.org/wiki/Multiple_dispatch">Multiple dispatch</a>: providing ability to define function behavior across many combinations of argument types</li>
<li><a class="mw-redirect" title="Dynamic type" href="https://en.wikipedia.org/wiki/Dynamic_type">Dynamic type</a>&nbsp;system: types for documentation, optimization, and dispatch</li>
<li>Good performance, approaching that of&nbsp;<a title="Type system" href="https://en.wikipedia.org/wiki/Type_system">statically-typed</a>&nbsp;languages like C</li>
<li>A built-in&nbsp;<a title="Package manager" href="https://en.wikipedia.org/wiki/Package_manager">package manager</a></li>
<li><a title="Lisp (programming language)" href="https://en.wikipedia.org/wiki/Lisp_(programming_language)">Lisp</a>-like macros and other&nbsp;<a title="Metaprogramming" href="https://en.wikipedia.org/wiki/Metaprogramming">metaprogramming</a>&nbsp;facilities</li>
<li>Call&nbsp;<a title="Python (programming language)" href="https://en.wikipedia.org/wiki/Python_(programming_language)">Python</a>&nbsp;functions: use the PyCall package</li>
<li>Call&nbsp;<a title="C (programming language)" href="https://en.wikipedia.org/wiki/C_(programming_language)">C</a>&nbsp;functions directly: no wrappers or special&nbsp;<a title="Application programming interface" href="https://en.wikipedia.org/wiki/Application_programming_interface">APIs</a></li>
<li>Powerful&nbsp;<a title="Shell (computing)" href="https://en.wikipedia.org/wiki/Shell_(computing)">shell</a>-like abilities to manage other processes</li>
<li>Designed for&nbsp;<a title="Parallel computing" href="https://en.wikipedia.org/wiki/Parallel_computing">parallel</a>&nbsp;and&nbsp;<a title="Distributed computing" href="https://en.wikipedia.org/wiki/Distributed_computing">distributed computing</a></li>
<li><a title="Coroutine" href="https://en.wikipedia.org/wiki/Coroutine">Coroutines</a>: lightweight&nbsp;<a title="Green threads" href="https://en.wikipedia.org/wiki/Green_threads"><em>green</em>&nbsp;threading</a></li>
<li>User-defined types are as fast and compact as built-ins</li>
<li>Automatic generation of efficient, specialized code for different argument types</li>
<li>Elegant and extensible conversions and promotions for numeric and other types</li>
<li>Efficient support for&nbsp;<a title="Unicode" href="https://en.wikipedia.org/wiki/Unicode">Unicode</a>, including but not limited to&nbsp;<a title="UTF-8" href="https://en.wikipedia.org/wiki/UTF-8">UTF-8</a></li>
</ul>
<p>Multiple dispatch (also termed&nbsp;<a class="mw-redirect" title="Multimethod" href="https://en.wikipedia.org/wiki/Multimethod">multimethods</a>&nbsp;in Lisp) is a&nbsp;<a title="Generalization" href="https://en.wikipedia.org/wiki/Generalization">generalization</a>&nbsp;of&nbsp;<a class="mw-redirect" title="Single dispatch" href="https://en.wikipedia.org/wiki/Single_dispatch">single dispatch</a>&nbsp;&ndash; the&nbsp;<a title="Polymorphism (computer science)" href="https://en.wikipedia.org/wiki/Polymorphism_(computer_science)">polymorphic mechanism</a>&nbsp;used in common&nbsp;<a title="Object-oriented programming" href="https://en.wikipedia.org/wiki/Object-oriented_programming">object-oriented programming</a>&nbsp;(OOP) languages&nbsp;&ndash; that uses&nbsp;<a title="Inheritance (object-oriented programming)" href="https://en.wikipedia.org/wiki/Inheritance_(object-oriented_programming)">inheritance</a>. In Julia, all concrete types are&nbsp;<a title="Subtyping" href="https://en.wikipedia.org/wiki/Subtyping">subtypes</a>&nbsp;of abstract types, directly or indirectly subtypes of the&nbsp;<em>Any</em>&nbsp;type, which is the top of the type hierarchy. Concrete types can not themselves be subtyped the way they can in other languages; composition is used instead (see also&nbsp;<a title="Inheritance (object-oriented programming)" href="https://en.wikipedia.org/wiki/Inheritance_(object-oriented_programming)#Inheritance_vs_subtyping">inheritance vs subtyping</a>).</p>
<p>Julia draws significant inspiration from various dialects of Lisp, including&nbsp;<a title="Scheme (programming language)" href="https://en.wikipedia.org/wiki/Scheme_(programming_language)">Scheme</a>&nbsp;and&nbsp;<a title="Common Lisp" href="https://en.wikipedia.org/wiki/Common_Lisp">Common Lisp</a>, and it shares many features with&nbsp;<a title="Dylan (programming language)" href="https://en.wikipedia.org/wiki/Dylan_(programming_language)">Dylan</a>, also a multiple-dispatch-oriented dynamic language (which features an&nbsp;<a title="ALGOL" href="https://en.wikipedia.org/wiki/ALGOL">ALGOL</a>-like&nbsp;<a title="Free-form language" href="https://en.wikipedia.org/wiki/Free-form_language">free-form</a>&nbsp;<a title="Infix notation" href="https://en.wikipedia.org/wiki/Infix_notation">infix</a>&nbsp;syntax rather than a Lisp-like prefix syntax, while in Julia "everything"<sup id="cite_ref-Learn_Julia_in_Y_79-0" class="reference"><a href="https://en.wikipedia.org/wiki/Julia_(programming_language)#cite_note-Learn_Julia_in_Y-79">[77]</a></sup>&nbsp;is an&nbsp;<a title="Expression (computer science)" href="https://en.wikipedia.org/wiki/Expression_(computer_science)">expression</a>), and with&nbsp;<a title="Fortress (programming language)" href="https://en.wikipedia.org/wiki/Fortress_(programming_language)">Fortress</a>, another numerical programming language (which features multiple dispatch and a sophisticated parametric type system). While&nbsp;<a title="Common Lisp Object System" href="https://en.wikipedia.org/wiki/Common_Lisp_Object_System">Common Lisp Object System</a>&nbsp;(CLOS) adds multiple dispatch to Common Lisp, not all functions are generic functions.</p>
<p>In Julia, Dylan, and Fortress extensibility is the default, and the system's built-in functions are all generic and extensible. In Dylan, multiple dispatch is as fundamental as it is in Julia: all user-defined functions and even basic built-in operations like&nbsp;<code>+</code>&nbsp;are generic. Dylan's type system, however, does not fully support parametric types, which are more typical of the&nbsp;<a title="Generational list of programming languages" href="https://en.wikipedia.org/wiki/Generational_list_of_programming_languages#ML_based">ML lineage of languages</a>. By default, CLOS does not allow for dispatch on Common Lisp's parametric types; such extended dispatch semantics can only be added as an extension through the&nbsp;<a title="Common Lisp Object System" href="https://en.wikipedia.org/wiki/Common_Lisp_Object_System#Metaobject_Protocol">CLOS Metaobject Protocol</a>. By convergent design, Fortress also features multiple dispatch on parametric types; unlike Julia, however, Fortress is statically rather than dynamically typed, with separate compiling and executing phases. <p>By default, the Julia runtime must be pre-installed as user-provided source code is run. Alternatively, a standalone&nbsp;<a title="Executable" href="https://en.wikipedia.org/wiki/Executable">executable</a>&nbsp;that needs no Julia source code can be built with&nbsp;<em>ApplicationBuilder.jl</em>&nbsp;and&nbsp;<em>PackageCompiler.jl</em>.</p>
<p>Julia's&nbsp;<a title="Macro (computer science)" href="https://en.wikipedia.org/wiki/Macro_(computer_science)#Syntactic_macros">syntactic macros</a>&nbsp;(used for&nbsp;<a title="Metaprogramming" href="https://en.wikipedia.org/wiki/Metaprogramming">metaprogramming</a>), like Lisp macros, are more powerful than&nbsp;<a title="Macro (computer science)" href="https://en.wikipedia.org/wiki/Macro_(computer_science)#Text-substitution_macros">text-substitution macros</a>&nbsp;used in the&nbsp;<a title="Preprocessor" href="https://en.wikipedia.org/wiki/Preprocessor">preprocessor</a>&nbsp;of some other languages such as C, because they work at the level of&nbsp;<a title="Abstract syntax tree" href="https://en.wikipedia.org/wiki/Abstract_syntax_tree">abstract syntax trees</a>&nbsp;(ASTs). Julia's macro system is&nbsp;<a title="Hygienic macro" href="https://en.wikipedia.org/wiki/Hygienic_macro">hygienic</a>, but also supports deliberate capture when desired (like for&nbsp;<a title="Anaphoric macro" href="https://en.wikipedia.org/wiki/Anaphoric_macro">anaphoric macros</a>) using the&nbsp;<code>esc</code>&nbsp;construct.</p>
<h2><span id="Interaction" class="mw-headline">Interaction</span></h2>
<p>The Julia official distribution includes a "full-featured interactive command-line REPL" (<a title="Read&ndash;eval&ndash;print loop" href="https://en.wikipedia.org/wiki/Read%E2%80%93eval%E2%80%93print_loop">read&ndash;eval&ndash;print loop</a>),&nbsp;with a searchable history,&nbsp;<a class="mw-redirect" title="Tab-completion" href="https://en.wikipedia.org/wiki/Tab-completion">tab-completion</a>, many helpful keybindings, and dedicated help and&nbsp;<a title="Shell (computing)" href="https://en.wikipedia.org/wiki/Shell_(computing)">shell</a>&nbsp;modes;&nbsp;which can be used to experiment and test code quickly.&nbsp;The following fragment represents a sample session example where strings are concatenated automatically by println:</p>
<div class="mw-highlight mw-highlight-lang-jlcon mw-content-ltr" dir="ltr">
<pre><span class="gp">julia&gt;</span> <span class="n">p</span><span class="p">(</span><span class="n">x</span><span class="p">)</span> <span class="o">=</span> <span class="mi">2</span><span class="n">x</span><span class="o">^</span><span class="mi">2</span> <span class="o">+</span> <span class="mi">1</span><span class="p">;</span> <span class="n">f</span><span class="p">(</span><span class="n">x</span><span class="p">,</span> <span class="n">y</span><span class="p">)</span> <span class="o">=</span> <span class="mi">1</span> <span class="o">+</span> <span class="mi">2</span><span class="n">p</span><span class="p">(</span><span class="n">x</span><span class="p">)</span><span class="n">y</span>
<span class="gp">julia&gt;</span> <span class="n">println</span><span class="p">(</span><span class="s">"Hello world!"</span><span class="p">,</span> <span class="s">" I'm on cloud "</span><span class="p">,</span> <span class="n">f</span><span class="p">(</span><span class="mi">0</span><span class="p">,</span> <span class="mi">4</span><span class="p">),</span> <span class="s">" as Julia supports recognizable syntax!"</span><span class="p">)</span>
<span class="go">Hello world! I'm on cloud 9 as Julia supports recognizable syntax!</span>
</pre>
</div>
<p>The REPL gives user access to the system shell and to help mode, by pressing&nbsp;<code>;</code>&nbsp;or&nbsp;<code>?</code>&nbsp;after the prompt (preceding each command), respectively. It also keeps the history of commands, including between sessions.&nbsp;Code that can be tested inside the Julia's interactive section or saved into a file with a&nbsp;<code>.jl</code>&nbsp;extension and run from the command line by typing:</p>

<p>Julia is supported by&nbsp;<a class="mw-redirect" title="Jupyter" href="https://en.wikipedia.org/wiki/Jupyter">Jupyter</a>, an online interactive "notebooks" environment.</p>
<h3><span id="Use_with_other_languages" class="mw-headline">Use with other languages</span></h3>
<p>Julia is in practice&nbsp;<a title="Interoperability" href="https://en.wikipedia.org/wiki/Interoperability">interoperable</a>&nbsp;with many languages. Julia's&nbsp;<code class="mw-highlight mw-highlight-lang-julia" dir="ltr"><span class="k">ccall</span></code>&nbsp;keyword is used to call C-exported or Fortran shared library functions individually.</p>
<p>Julia has support for&nbsp;<a class="mw-redirect" title="Unicode 12.1" href="https://en.wikipedia.org/wiki/Unicode_12.1">Unicode 12.1</a>&nbsp;(support for current Unicode 13.0 has been implemented&nbsp;and will for Julia 1.5), which added only one letter since Unicode 12.0, with&nbsp;<a title="UTF-8" href="https://en.wikipedia.org/wiki/UTF-8">UTF-8</a>&nbsp;used for strings (by default) and for Julia source code (only allowing legal UTF-8 in latest version), meaning also allowing as an option common math symbols for many operators, such as &isin; for the&nbsp;<code>in</code>&nbsp;operator.</p>
<p>Julia has packages supporting markup languages such as&nbsp;<a title="HTML" href="https://en.wikipedia.org/wiki/HTML">HTML</a>&nbsp;(and also for&nbsp;<a class="mw-redirect" title="HTTP" href="https://en.wikipedia.org/wiki/HTTP">HTTP</a>),&nbsp;<a title="XML" href="https://en.wikipedia.org/wiki/XML">XML</a>,&nbsp;<a title="JSON" href="https://en.wikipedia.org/wiki/JSON">JSON</a>&nbsp;and&nbsp;<a title="BSON" href="https://en.wikipedia.org/wiki/BSON">BSON</a>, and for&nbsp;<a title="Database" href="https://en.wikipedia.org/wiki/Database">databases</a>&nbsp;and web use in general.</p>
<p>&nbsp;</p>
<h2><span id="Usage" class="mw-headline">Usage</span></h2>
<p>Libraries such as&nbsp;<a title="Flux (machine-learning framework)" href="https://en.wikipedia.org/wiki/Flux_(machine-learning_framework)">Flux</a>&nbsp;and&nbsp;<a class="new" title="Knet (machine-learning framework) (page does not exist)" href="https://en.wikipedia.org/w/index.php?title=Knet_(machine-learning_framework)&amp;action=edit&amp;redlink=1">Knet</a>&nbsp;implement complete machine-learning frameworks written completely in Julia.&nbsp;<a title="Apache MXNet" href="https://en.wikipedia.org/wiki/Apache_MXNet">Apache MXNet</a>&nbsp;and&nbsp;<a title="TensorFlow" href="https://en.wikipedia.org/wiki/TensorFlow">TensorFlow</a>&nbsp;are also available through wrapper libraries.&nbsp;<a class="new" title="Turing (probabilistic programming) (page does not exist)" href="https://en.wikipedia.org/w/index.php?title=Turing_(probabilistic_programming)&amp;action=edit&amp;redlink=1">Turing</a>&nbsp;implements a general-purpose&nbsp;<a title="Probabilistic programming" href="https://en.wikipedia.org/wiki/Probabilistic_programming">probabilistic programming</a>&nbsp;language into Julia.&nbsp;<a title="Pharmacokinetics" href="https://en.wikipedia.org/wiki/Pharmacokinetics">Pharmacokinetic</a>&nbsp;modeling and simulations are made available through the&nbsp;<a class="new" title="Pumas (pharmaceutical modeling) (page does not exist)" href="https://en.wikipedia.org/w/index.php?title=Pumas_(pharmaceutical_modeling)&amp;action=edit&amp;redlink=1">Pumas</a>&nbsp;suite.</p>
<h2><span id="Implementation" class="mw-headline">Implementation</span></h2>
<p>Julia's core is implemented in Julia and&nbsp;<a title="C (programming language)" href="https://en.wikipedia.org/wiki/C_(programming_language)">C</a>, together with&nbsp;<a title="C++" href="https://en.wikipedia.org/wiki/C%2B%2B">C++</a>&nbsp;for the&nbsp;<a title="LLVM" href="https://en.wikipedia.org/wiki/LLVM">LLVM</a>&nbsp;dependency. The parsing and code-lowering are implemented in FemtoLisp, a&nbsp;<a title="Scheme (programming language)" href="https://en.wikipedia.org/wiki/Scheme_(programming_language)">Scheme</a>&nbsp;dialect.&nbsp;The LLVM compiler infrastructure project is used as the&nbsp;<a title="Compiler" href="https://en.wikipedia.org/wiki/Compiler#Back_end">back end</a>&nbsp;for generation of&nbsp;<a title="64-bit computing" href="https://en.wikipedia.org/wiki/64-bit_computing">64-bit</a>&nbsp;or&nbsp;<a class="mw-redirect" title="32-bit" href="https://en.wikipedia.org/wiki/32-bit">32-bit</a>&nbsp;optimized&nbsp;<a title="Machine code" href="https://en.wikipedia.org/wiki/Machine_code">machine code</a>&nbsp;depending on the platform Julia runs on. With some exceptions (e.g.,&nbsp;<a class="mw-redirect" title="PCRE" href="https://en.wikipedia.org/wiki/PCRE">PCRE</a>), the&nbsp;<a title="Standard library" href="https://en.wikipedia.org/wiki/Standard_library">standard library</a>&nbsp;is implemented in Julia itself. The most notable aspect of Julia's implementation is its speed, which is often within a factor of two relative to fully optimized C code (and thus often an order of magnitude faster than Python or&nbsp;<a title="R (programming language)" href="https://en.wikipedia.org/wiki/R_(programming_language)">R</a>).&nbsp;Development of Julia began in 2009 and an&nbsp;<a title="Open-source software" href="https://en.wikipedia.org/wiki/Open-source_software">open-source</a>&nbsp;version was publicized in February 2012.</p>
<h3><span id="Current_and_future_platforms" class="mw-headline">Current and future platforms</span></h3>
<p>While Julia uses JIT, Julia generates native machine code directly, before a function is first run (not&nbsp;<a title="Bytecode" href="https://en.wikipedia.org/wiki/Bytecode">bytecodes</a>&nbsp;that are run on a&nbsp;<a title="Virtual machine" href="https://en.wikipedia.org/wiki/Virtual_machine">virtual machine</a>&nbsp;(VM) or translated as the bytecode is running, as with,&nbsp;<em>e.g.</em>, Java; the&nbsp;<a class="mw-redirect" title="JVM" href="https://en.wikipedia.org/wiki/JVM">JVM</a>&nbsp;or&nbsp;<a title="Dalvik (software)" href="https://en.wikipedia.org/wiki/Dalvik_(software)">Dalvik</a>&nbsp;in Android).</p>
<p>Julia has four support tiers,&nbsp;and currently supports all&nbsp;<a title="X86-64" href="https://en.wikipedia.org/wiki/X86-64">x86-64</a>&nbsp;processors, that are&nbsp;<a title="64-bit computing" href="https://en.wikipedia.org/wiki/64-bit_computing">64-bit</a>&nbsp;(and is more&nbsp;<a title="Program optimization" href="https://en.wikipedia.org/wiki/Program_optimization">optimized</a>&nbsp;for the latest generations) and all&nbsp;<a title="IA-32" href="https://en.wikipedia.org/wiki/IA-32">IA-32</a>&nbsp;("x86") processors except for decades old ones, i.e., in&nbsp;<a class="mw-redirect" title="32-bit" href="https://en.wikipedia.org/wiki/32-bit">32-bit</a>&nbsp;mode ("i686", excepting CPUs from the pre-<a title="Pentium 4" href="https://en.wikipedia.org/wiki/Pentium_4">Pentium 4</a>-era); and supports more in lower tiers, e.g., ARM has tier 2 support: Julia "fully supports ARMv8 (AArch64) processors, and supports ARMv7 and ARMv6 (AArch32) with some caveats."&nbsp;<a title="CUDA" href="https://en.wikipedia.org/wiki/CUDA">CUDA</a>&nbsp;(i.e. Nvidia GPUs; implementing&nbsp;<a title="Parallel Thread Execution" href="https://en.wikipedia.org/wiki/Parallel_Thread_Execution">PTX</a>) has tier 1 support, with the help of an&nbsp;<a class="external text" href="https://github.com/JuliaGPU/CUDAnative.jl" rel="nofollow">external package</a>. There are also additionally packages supporting other accelerators, such as Google's&nbsp;<a title="Tensor processing unit" href="https://en.wikipedia.org/wiki/Tensor_processing_unit">TPUs</a>,&nbsp;and AMD's GPUs also have support with e.g.&nbsp;<a title="OpenCL" href="https://en.wikipedia.org/wiki/OpenCL">OpenCL</a>. Julia's downloads page provides&nbsp;<a title="Executable" href="https://en.wikipedia.org/wiki/Executable">executables</a>&nbsp;(and source) for all the officially supported platforms.</p>
<p>On some platforms, Julia may need to be compiled from&nbsp;<a title="Source code" href="https://en.wikipedia.org/wiki/Source_code">source code</a>&nbsp;(e.g., the original&nbsp;<a title="Raspberry Pi" href="https://en.wikipedia.org/wiki/Raspberry_Pi">Raspberry Pi</a>), with specific build options. Julia has been "successfully built" on several ARM platforms, up to, e.g., "ARMv8 Data Center &amp; Cloud Processors", such as&nbsp;<a class="external text" href="https://www.cavium.com/ThunderX_ARM_Processors.html" rel="nofollow">Cavium ThunderX</a>&nbsp;(first ARM with 48 cores). PowerPC (64-bit) has tier 3 support meaning "may or may not build". Julia is now supported in&nbsp;<a title="Raspbian" href="https://en.wikipedia.org/wiki/Raspbian">Raspbian</a>&nbsp;while support is better for newer Pis, e.g. those with ARMv7 or newer; the Julia support is promoted by the&nbsp;<a title="Raspberry Pi Foundation" href="https://en.wikipedia.org/wiki/Raspberry_Pi_Foundation">Raspberry Pi Foundation</a>.&nbsp;There's also support for web browsers/<a title="JavaScript" href="https://en.wikipedia.org/wiki/JavaScript">JavaScript</a>&nbsp;through JSExpr.jl;&nbsp;and the alternative language of web browsers,&nbsp;<a title="WebAssembly" href="https://en.wikipedia.org/wiki/WebAssembly">WebAssembly</a>, has support&nbsp;through minimal support in Julia itself, that supports several upcoming external Julia projects. Julia can compile to ARM; thus in theory&nbsp;<a class="mw-redirect" title="Android app" href="https://en.wikipedia.org/wiki/Android_app">Android apps</a>&nbsp;can be made, but for now only Julia itself (and most packages) has been made to run under&nbsp;<a title="Android (operating system)" href="https://en.wikipedia.org/wiki/Android_(operating_system)">Android</a>&nbsp;indirectly, i.e. in Ubuntu on Android.</p>
